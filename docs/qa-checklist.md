# QA Checklist

## 结构检查

- [ ] `index.html` 有顶层 `data-composition-id="main"`
- [ ] video 是 `muted playsinline`
- [ ] audio 是单独 `<audio>`
- [ ] 所有字幕 / 章节 / 解释层都有 `data-start`
- [ ] 所有字幕 / 章节 / 解释层都有 `data-duration`
- [ ] 所有 timed clip 都有唯一合理的 `data-track-index`
- [ ] timeline 同步注册：`window.__timelines["main"]`

## 剪辑切点

- [ ] 每个 EDL 保留段都有语义理由。
- [ ] 每个切点都生成电影条 + 波形。
- [ ] 逐张检查嘴型、动作、音节截断、重复语义和过紧节奏。
- [ ] 只有看完证据后才存在 `qa/cuts/approval.json`。

## 字幕检查

- [ ] 字幕校准先通篇读完整条转写，明确主题、术语体系、口播语境，再做局部修字。
- [ ] 不凭单个 ASR 片段或常识改核心术语；用户确认过的词优先于模型猜测。
- [ ] 平台名、模型名、产品绰号、行业词必须全片统一，不能一处一个写法。
- [ ] 字幕不是运行时 DOM mutation
- [ ] 字幕已从 JSON 生成静态 clip
- [ ] 字幕不长期压嘴
- [ ] 字幕字号在手机上足够大
- [ ] 英文产品名没有被切得太碎
- [ ] 解释层出现时，字幕保留并按需要让位

## 解释层检查

- [ ] 不挡眼睛
- [ ] 不长时间挡嘴
- [ ] 半透明，不像整页 PPT
- [ ] 信息少而准
- [ ] 每个解释层都在最终 MP4 抽帧里出现

## B-roll 检查

- [ ] 每段都有明确 `intent` 与 `reason`。
- [ ] 默认保留说话人小窗。
- [ ] 前 3 秒默认不用、单段 ≤10 秒、总占比 ≤25%、相互不重叠。
- [ ] 真实产品界面使用真实截图或录屏，不用伪造 UI。

## 抖音安全区

- [ ] 上方不贴太近
- [ ] 下方不被底部 UI / 字幕区吃掉
- [ ] 右侧留出互动栏空间
- [ ] 关键字不放在最右侧
- [ ] 人脸是主体，包装是辅助

## 最终文件检查

用 `ffprobe` 或 `npm run qa` 确认：

- [ ] 1080x1920
- [ ] 60fps
- [ ] 有音轨
- [ ] 时长正确
- [ ] 码率接近 24Mbps
- [ ] 最终 MP4 可以正常播放

## 多版本检查

- [ ] 抖音竖屏：1080x1920，字幕不压嘴，右侧互动栏安全。
- [ ] YouTube 横屏：1920x1080，口播主体保留，信息层不变 PPT。
- [ ] YouTube Shorts：从母版或高码率竖屏派生，不从平台下载件二压。
- [ ] 每个 variant 都单独跑 `lint / validate / inspect`。
- [ ] 每个 variant 都单独抽最终 MP4 帧。

## 抽帧检查

至少抽：

- [ ] 开头 0-10s
- [ ] 普通字幕点
- [ ] 章节出现点
- [ ] 每个解释层出现点
- [ ] 结尾 CTA

抽帧文件位置：

```text
jobs/<slug>/qa/final-frames/
```
