# Product

## Register

product

## Users

单人口播创作者。典型场景:拍完口播(常在夜里、暗光房间),坐在 Mac 前要把一条素材变成抖音竖屏 / YouTube 横屏 / Shorts 多版本成片。用户用 Claude Code 或 Codex 当智能入口,判断类工序(精剪、字幕校准、排拍)交给模型,但要一个稳定的控制面把确定性工序串起来。

## Product Purpose

口播工厂是创作者的视觉证据库与视频生产控制面。默认入口让人看见、比较、诊断和选择以前做过的全部主题、组件、布局、预设与历史视觉套装；视频项目入口继续承载"素材→剪辑→字幕→包装→出片"。组件的新增和修改交给 Codex、Claude 或 CLI，App 自动发现并用真实 HyperFrames 结果展示。成功 = 不打开代码或 JSON 也知道自己有什么、能否使用、竖横屏效果如何，以及坏在哪里。

## Brand Personality

暗房工作台、冷静、结果导向。像调色师的控制台而不是工程师的调试面板:仪器感、少即是多、成片画面永远是最亮的东西。

## Anti-references

- 工程师调试面板(一堆开关和日志,用户不知道自己在哪一步)。
- SaaS 仪表盘模板(hero 大数字 + 卡片矩阵 + 渐变强调)。
- 赛博朋克霓虹堆砌;控制台自身抢成片的戏。

## Design Principles

1. **展示先于配置**:先让全部视觉资产可见、可比较、可追溯，再谈编辑能力。
2. **生成归 Skill,展示归 App**:Codex、Claude 或 CLI 创建组件；App 不做拖拽编辑器。
3. **真实渲染是证据**:组件 poster、动态预览和最终 MP4 优先于状态灯；不使用近似 mock 冒充成片。
4. **坏资产不消失**:历史、损坏、过期和不支持的资产都保留，并说明原因、来源与替代项。
5. **竖横屏同等重要**:抖音 9:16 与 YouTube 16:9 都从同一 A-roll 母版重排，不互相二压。
6. **向导仍然稳定**:视频项目永远知道当前工位、下一步和运行状态。
7. **边界一句话**:全部在本机运行，素材与成片不出这台电脑。

## Accessibility & Inclusion

单用户桌面 Chrome 场景。仍守:文本对比 ≥ AA、键盘可达、focus 可见、`prefers-reduced-motion` 全覆盖、触达区 ≥ 40px。
